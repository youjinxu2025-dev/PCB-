# 发布说明

## 当前版本适合的发布方式

这版最适合发布到支持 Node.js 服务的云平台，例如 Render 或自己的云服务器。

原因：

- 网站不只是静态页面
- 包含登录、下单、上传、支付登记、后台查看
- 需要服务端 API 和可写入存储

## 发布前已处理

- 服务监听 `0.0.0.0`
- 端口支持读取 `PORT`
- 存储目录支持读取 `STORAGE_DIR`
- 密码改为哈希保存
- 增加健康检查接口：`/api/health`
- 增加 `package.json`

## Render 发布步骤

1. 把项目上传到 GitHub
2. 在 Render 新建 `Web Service`
3. 连接 GitHub 仓库
4. Render 会自动识别 `render.yaml`
5. 首次发布完成后得到一个公网地址

## 重要提醒

当前版本已经可以上线试运行，但仍然属于“第一版商用原型”：

- 数据还保存在 JSON 文件里，不适合高并发
- 上传文件还保存在服务磁盘中
- 支付还是站内登记，不是真实微信/支付宝接口

## 下一步建议

如果要正式长期商用，建议继续升级：

1. 把用户、订单、付款迁移到 Postgres
2. 把上传文件迁移到 S3 / R2
3. 接入真实支付接口
4. 增加管理员登录和权限控制
